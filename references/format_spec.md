# 格式规范（人类可读参考）

> ⚠️ 本文件仅供人工审阅。**唯一权威**是 `spec/format_spec.json`；脚本只读 JSON。
> 请勿让模型据本文档「理解并发挥」——判定逻辑全部在脚本里，阈值全部来自 JSON。

## 封皮
- 背景绿色；标注密级、项目单位、项目名称、考核年份。
- 应含字段：密级、项目名称、承担单位（盖章）、项目负责人（签字）、项目起止时间、报告编制时间。
- 题目：方正小标宋，20 磅，居中。
- （绿色背景 / 缺字段无法自动补 → 仅批注提示。）

## 目录
- 仿宋，**14 磅**；不超过三级标题。
- **按级缩进**（首行/右缩进一律清零，左缩进按级设为 `leftChars`）：
  | 目录层级 | 左缩进 | leftChars |
  |---|---|---|
  | 一级 | 无缩进 | 0 |
  | 二级 | 2 字符 | 200 |
  | 三级 | 4 字符 | 400 |
  层级取自目录条目样式（`TOC1/TOC2/TOC3`）末尾数字；只动 `w:ind`，不碰 TOC 域自身用来对齐点线号和
  页码的制表位 `w:tabs`。
- 目录是「域」，用户打开时会自动刷新（本 skill 设了 `updateFields`），刷新时 Word 会**按 TOC 样式重建
  条目**、丢弃直接格式。因此除了改当前条目的直接格式外，还会把 `TOC1/TOC2/TOC3` 等目录条目**样式本身**
  改成仿宋/14 磅/按级缩进，保证刷新后仍符合规范。

## 空行
- 空白/纯空格段落不参与题目/标题/正文/图表的任何判定，跳过。

## 页面设置（容差 ±3 twips）
| 项 | 规范 | twips |
|---|---|---|
| 上边距 | 3.5 cm | 1984 |
| 下边距 | 3.2 cm | 1814 |
| 左边距 | 2.85 cm | 1616 |
| 右边距 | 2.85 cm | 1616 |
| 页眉 | 1.5 cm | 850 |
| 页脚 | 1.75 cm | 992 |

## 行距
- 固定值 28 磅（`lineRule=exact`, `line=560`）。

## 字体字号（三号 = 16 磅 = 半磅值 32）
| 对象 | 中文字体 | 字号 | 首行缩进 |
|---|---|---|---|
| 题目 | 方正小标宋 | 20 磅 | 居中，无缩进 |
| 一级标题 | 黑体 | 三号 | 2 字符 |
| 二级标题 | 楷体 | 三号 | 2 字符 |
| 三级标题 | 仿宋 | 三号 | 2 字符 |
| 四级标题 | 仿宋 | 三号 | 2 字符 |
| 正文 | 仿宋 | 三号 | 2 字符 |
| 表格内容 | 仿宋 | 14 磅 | —（仅校验字体字号，不动缩进/行距） |
| 西文 | Times New Roman | 三号 | — |
| 页码 | Times New Roman | 小四（12 磅） | —（提示项） |

- 首行缩进 2 字符 = `firstLineChars=200`；修正时**同时清除左、右缩进**（`leftChars/left/startChars/start`
  与 `rightChars/right/endChars/end`），只保留首行缩进。因为原本若已有左缩进，首行会缩进「左缩进＋2 字符」
  实际大于 2 字符，所以左右缩进都要归零。
- 题目「无缩进」= 首行缩进清零、且左右缩进全部清除。只清左不清右（或反之）会让 `jc=center` 的居中基准
  偏移到「缩进后的可用宽度」而不是整个页面宽度，视觉上就不是真正居中——必须两侧都清。
- 清缩进用的是「显式置 0」而非「删除属性」：缩进常来自段落样式，仅删直接属性会露出样式里的缩进，所以
  统一把 `left/leftChars`（及 `right/rightChars`）直接写成 0 覆盖样式继承值。

## 图表
- 按顺序编号，并标注内容。
- 图表标题排版：**居中、无任何缩进**（首行/左/右缩进全部清零，`jc=center`）；字体字号规范未定义，不改。
- 图/表标题的判定**优先于**标题判定：只要文本形如「图/表+数字」，或段落样式名包含题注类关键词（题注、
  Caption、图表标题、表格标题、**图标题、表标题**、图题、表题等），一律先归为图表标题候选，即使该段落
  意外带有 `outlineLvl` 或样式名也含「标题」二字（例如「图表标题」「表标题」样式本身就带「标题」两个字），
  也不会被判成正文标题。
  - 特别地，`图标题`/`表标题` 必须显式列入题注样式关键词——较短的 `图题`/`表题` 并不是它们的子串
    （中间隔了个「标」字），否则「表标题」样式的段落会漏判成一级标题。
- 序号不连续 → 自动重编号（图/表各自计数；若原编号带「-/.」则按「章-序」分组计数）。
- 缺内容说明 → 仅批注提示。
- 题注类样式已确认、但**没有任何编号**（`num_raw` 为 null，例如「表标题」样式的段落只写了「设备清单」）：
  - **平铺编号**时 → 按顺序**自动补**「图N/表N 」编号（样式已确认，补号是安全的），并加批注说明。
  - **章-序分组编号**时 → 无法推断该条属于哪一章，仍仅批注提示可能漏编号，不臆造章号。

## 序号连续性
- 标题：一级/二级/三级/四级各自计数，父级递增时子级归零；目录段落不参与计数。
- 标题的**层级**先由脚本按形状/样式给出默认猜测，再经模型通读全文候选清单复核修正（正则无法保证覆盖
  每份文档的实际写法，例如同一份文档可能通篇用「1.」表示一级标题）；复核后的层级才用于本节的连续性计数
  与字体字号判定。
- 层级判定优先级：**大纲级别 `outlineLvl` > 样式名里写明的级别（「标题 1」「一级标题」等）> 序号形状**。
  即样式名说了是几级就按几级，不会被序号形状带偏——所以一级标题写成「1. 项目进展」不会被当成三级
  （「数字.」形状默认猜三级），而是按一级把序号规范化为「一、项目进展」；样式正确但序号遗漏/错误/不连续
  的，只要有大纲级别或标题样式撑腰，就**直接改**、不需人工确认。
- **确认级别决定是否自动改**：有 `outlineLvl`/标题样式撑腰、或经模型复核确认的标题/图表标题，序号有误
  时直接改；仅靠「形如编号」的文本形状匹配、没有任何样式/大纲级别支撑的，不计入连续性计数、也不自动改，
  只生成提示批注让人工判断是否为真标题/图表标题。
- 不连续处（含原编号数字系统用错，例如一级标题误用阿拉伯数字）自动改为应有序号，并加批注。
- **序号分隔符**：序号与标题正文之间除「、.．」和括号外，也可能只用**空格或制表符**分隔
  （如「5 项目运行管理情况」「2⇥背景」）。这类序号同样能被识别并按位置改正（上例中错误的「5」→「二」）。
  为避免把年份等误当序号，空格/制表符分隔的形式只接受 **1–2 位阿拉伯数字或中文数字**（「2024 年度总结」
  的 2024 是四位，不会被当序号）；且此判定只作用于已确认样式的标题的序号解析，不会把正文里偶然以数字
  开头的段落变成标题。
- 识别为标题但确实没有任何编号 → 不臆造编号，仅批注提示可能漏编号（图/表标题在平铺编号时会自动补号，
  见「图表」一节）。
